SHA-256 (Secure Hash Algorithm 256-bit) - это криптографическая хэш-функция из семейства SHA-2. 
Она принимает на вход сообщение произвольной длины и вычисляет фиксированный хэш-значение размером 256 бит (32 байта). 
Хэш-значение обычно представляется в виде шестнадцатеричной строки.
SHA-256 обеспечивает следующие свойства:
Устойчивость к коллизиям: Для разных входных сообщений крайне маловероятно получить одинаковые хэш-значения.
Необратимость: Невозможно восстановить исходное сообщение по его хэш-значению.
Детерминированность: Для одного и того же входного сообщения всегда будет вычисляться одинаковое хэш-значение.
Чувствительность к изменениям: Даже небольшое изменение входных данных приводит к значительному изменению хэш-значения.
SHA-256 широко используется в различных областях, включая криптографию, безопасность данных и цифровые подписи. 
Его применения включают проверку целостности данных, хранение паролей в зашифрованном виде, цифровые сертификаты, блокчейн и многое другое.

-------Генерация ключей:---------
Выбираются два простых больших числа p и q.
Вычисляется их произведение n = p * q, которое становится модулем для шифрования и расшифровки.
Вычисляется значение функции Эйлера от n: phi(n) = (p - 1) * (q - 1).
Выбирается целое число e, взаимно простое с phi(n) и меньшее phi(n). Значение e становится публичным ключом для шифрования.
Вычисляется число d, обратное e по модулю phi(n), то есть d * e ≡ 1 (mod phi(n)). 
Значение d становится приватным ключом для расшифровки или создания подписи.
-------Шифрование данных:--------
Публичный ключ {e, n} используется для преобразования исходных данных в зашифрованное сообщение. 
Зашифрование выполняется по формуле: ciphertext = plaintext^e mod n, где plaintext - исходные данные, ciphertext - зашифрованное сообщение.
-------Расшифровка данных:--------
Приватный ключ d используется для расшифровки зашифрованного сообщения ciphertext. 
Расшифровка выполняется по формуле: plaintext = ciphertext^d mod n, где ciphertext - зашифрованное сообщение, plaintext - исходные данные.
-------Цифровая подпись:--------
Приватный ключ d используется для создания цифровой подписи исходных данных. 
Подписание выполняется путем вычисления хэш-значения исходных данных, а затем применения операции шифрования с использованием приватного ключа d. 
Полученное зашифрованное значение является цифровой подписью.
При проверке подписи используется публичный ключ {e, n} для расшифровки цифровой подписи и сравнения с вычисленным хэш-значением исходных данных. 
Если значения совпадают, подпись считается действительной.
RSA широко применяется в области криптографии для защиты конфиденциальности данных, обеспечения аутентификации и создания цифровых подписей. 
Он является одним из наиболее распространенных алгоритмов для шифрования и подписи данных.
